# Python `==` 연산자와 `is` 연산자의 차이점

## 1. 핵심 요약

| 연산자   | 비교 대상         | 기준     | 설명                                               |
| :------- | :---------------- | :------- | :------------------------------------------------- |
| **`==`** | **값 (Value)**    | Equality | 두 객체 내부의 데이터가 같은지 확인                |
| **`is`** | **객체 (Object)** | Identity | 두 변수가 메모리상의 동일한 객체를 가리키는지 확인 |

---

## 2. 상세 비교

### `==` (Equality)

- 객체의 값이 논리적으로 동일한지 비교합니다.
- 내부적으로 `__eq__` 메서드를 호출하여 판단합니다.
- 서로 다른 메모리 주소에 있더라도 담고 있는 내용(데이터)이 같으면 `True`를 반환합니다.

### `is` (Identity)

- 객체의 정체성(Identity), 즉 메모리 주소가 동일한지 비교합니다.
- 내부적으로 `id()` 함수의 결과값을 비교하는 것과 같습니다.
- 두 변수가 정확히 같은 물리적 객체를 참조하고 있을 때만 `True`를 반환합니다.

---

## 3. 코드 예시

```python
# 동일한 값을 가진 서로 다른 리스트 생성
list_a = [1, 2, 3]
list_b = [1, 2, 3]
list_c = list_a

# 1. 값 비교 (==)
print(list_a == list_b)  # True (내용물이 같음)

# 2. 객체 비교 (is)
print(list_a is list_b)  # False (내용은 같으나 메모리 주소가 다름)
print(list_a is list_c)  # True (list_c가 list_a의 주소를 그대로 할당받음)
```
